┌────────────────────────────────────────────────────────────────────────────┐
│  ❌ PROBLÈME IDENTIFIÉ                                                     │
└────────────────────────────────────────────────────────────────────────────┘

  Le système de licence se basait sur DateTime.now() qui utilise l'horloge
  système locale. Un utilisateur pouvait simplement reculer la date de sa
  machine pour prolonger indéfiniment sa licence.

  Exemple:
    • Licence expire le: 2025-04-01
    • Utilisateur recule à: 2025-01-01
    • Résultat: Licence "valide" indéfiniment ❌


┌────────────────────────────────────────────────────────────────────────────┐
│  ✅ SOLUTION IMPLÉMENTÉE                                                   │
└────────────────────────────────────────────────────────────────────────────┘

  SecureTimeService - Service de validation du temps avec 3 niveaux:

    1. NTP (Network Time Protocol)
       → Obtient l'heure réelle depuis serveurs publics
       → Impossible à manipuler par l'utilisateur

    2. Détection de manipulation
       → Stocke la dernière heure vérifiée
       → Détecte les retours en arrière de l'horloge

    3. Stockage multi-niveaux
       → FlutterSecureStorage (chiffré)
       → SharedPreferences (backup)
       → Compteur de sessions (détection réinstallation)

┌────────────────────────────────────────────────────────────────────────────┐
│  💻 UTILISATION DANS LE CODE                                               │
└────────────────────────────────────────────────────────────────────────────┘

  Le LicenseService utilise DÉJÀ automatiquement le SecureTimeService.
  Aucune modification supplémentaire n'est nécessaire!

  Mais si vous voulez l'utiliser directement:

  // Initialisation
  final secureTimeService = SecureTimeService();
  await secureTimeService.initialize();

  // Obtenir l'heure sécurisée
  final timeResult = await secureTimeService.getSecureTime();

  // Valider une licence
  if (license.isExpiredSecure(timeResult.trustedTime)) {
    showExpiredDialog();
  }

  // Gérer les erreurs
  try {
    final timeResult = await secureTimeService.getSecureTime();
  } on TimeValidationException catch (e) {
    // Manipulation détectée!
    showBlockedDialog(e.message);
  }

┌────────────────────────────────────────────────────────────────────────────┐
│  ❓ FAQ RAPIDE                                                             │
└────────────────────────────────────────────────────────────────────────────┘

  Q: Que se passe-t-il sans Internet?
  R: Le service utilise le cache NTP (valide 24h) et calcule le temps
     écoulé. Il détecte toujours les retours en arrière.

  Q: L'utilisateur peut-il contourner?
  R: Très difficile. Il faudrait bloquer tous les serveurs NTP, supprimer
     le stockage chiffré, et réinstaller sans laisser de traces.

  Q: Impact sur les performances?
  R: Minimal. Premier appel: ~1s (NTP), puis cache: <10ms.

  Q: Compatible avec quels OS?
  R: Windows, Linux, macOS, Android, iOS.
