BONNE NOUVELLE! ✅
==================

Le script de migration est 100% COMPATIBLE avec les optimisations!

Mieux encore, il DÉTECTE AUTOMATIQUEMENT la version optimisée
et conserve tous les avantages de performance!


NOUVEAU SCRIPT DE MIGRATION
============================

Utilisez maintenant:
  migrer-client-existant-optimise.bat

Au lieu de:
  migrer-client-existant.bat (ancien)


AVANTAGES DU NOUVEAU SCRIPT
============================

✅ Détecte automatiquement la version optimisée
✅ Conserve Prisma pré-généré (pas de régénération)
✅ Conserve la DB template (si présente)
✅ Démarrage rapide après migration (7-9s)
✅ Compatible avec version standard (rétrocompatible)
✅ Messages détaillés sur la version installée


COMMENT MIGRER UN CLIENT
=========================

1. CHEZ VOUS (Développeur):
   
   a) Créer le package optimisé:
      preparer-pour-client-optimise.bat
   
   b) Copier le script de migration:
      copy migrer-client-existant-optimise.bat release\LOGESCO-Client-Optimise\
   
   c) Compresser et distribuer:
      Créer LOGESCO-Client-Optimise.zip


2. CHEZ LE CLIENT:
   
   a) Extraire le package:
      Décompresser LOGESCO-Client-Optimise.zip
   
   b) Sauvegarder les données:
      sauvegarder-donnees-client.bat
   
   c) Migrer:
      migrer-client-existant-optimise.bat
   
   d) Tester:
      DEMARRER-LOGESCO.bat


DÉTECTION AUTOMATIQUE
======================

Le script cherche la nouvelle version dans cet ordre:

1. LOGESCO-Client-Optimise\backend (RECOMMANDÉ!)
2. Package-Mise-A-Jour\LOGESCO-Client-Optimise\backend
3. Package-Mise-A-Jour\LOGESCO-Client-Ultimate\backend
4. LOGESCO-Client-Ultimate\backend
5. dist-portable

Si version optimisée détectée:
  ✅ Prisma Client pre-genere detecte
  ✅ Base de donnees template detectee
  ✅ Demarrage ultra-rapide apres migration!


DONNÉES CONSERVÉES
==================

Le script conserve TOUTES les données:

✅ Utilisateurs et mots de passe
✅ Rôles et permissions
✅ Produits et catégories
✅ Stock et mouvements
✅ Clients et fournisseurs

SAUVEGARDES CRÉÉES
==================

Le script crée plusieurs sauvegardes:

1. sauvegarde_client_YYYYMMDD_HHMMSS/
   (Sauvegarde complète avant migration)

2. backend_ancien/
   (Ancien backend complet)

3. backend/database/logesco_avant_migration.db
   (Base de données avant migration)


RESTAURATION SI PROBLÈME
=========================

Si la migration échoue:

  restaurer-ancienne-version.bat

Ou manuellement:
  1. taskkill /f /im node.exe
  2. rmdir /s /q backend
  3. ren backend_ancien backend
  4. DEMARRER-LOGESCO.bat

TEST LOCAL RECOMMANDÉ
=====================

Avant de migrer chez un client, testez localement:

1. Créer une copie de test:
   mkdir C:\LOGESCO-TEST
   xcopy /E /I C:\LOGESCO-PROD\* C:\LOGESCO-TEST\

2. Copier le nouveau package:
   xcopy /E /I release\LOGESCO-Client-Optimise C:\LOGESCO-TEST\LOGESCO-Client-Optimise\

3. Tester la migration:
   cd C:\LOGESCO-TEST
   sauvegarder-donnees-client.bat
   migrer-client-existant-optimise.bat

4. Vérifier:
   DEMARRER-LOGESCO.bat
