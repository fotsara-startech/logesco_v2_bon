PROBLÈME IDENTIFIÉ:
-------------------
Le package "LOGESCO-Client-Optimise" contient un client Prisma
qui a été PRÉ-GÉNÉRÉ avec une base de données VIERGE.

Même après restauration de vos données, Prisma utilise l'ancien
client qui ne "voit" rien.


SYMPTÔMES:
----------
✓ Migration réussit sans erreur
✓ Base de données contient les données (vérifié avec SQLite)
✗ Backend affiche: produits: 0, clients: 0, ventes: 0
✗ Application ne montre aucune donnée


CAUSE RACINE:
-------------
Package Optimisé
├── backend/node_modules/@prisma/client/  ← PRÉ-GÉNÉRÉ (base vide)
└── backend/database/logesco.db           ← Base vierge

Quand vous restaurez VOTRE base:
→ Le fichier logesco.db contient vos données
→ Mais Prisma utilise le client pré-généré (base vide)
→ Résultat: Prisma "voit" 0 enregistrements
