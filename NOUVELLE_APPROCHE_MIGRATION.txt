MIGRATION DU CODE (TOUJOURS):
==============================

Utilisez: MIGRATION-SIMPLE.bat

Ce script:
✅ Met à jour le Backend (nouveau code)
✅ Met à jour le Frontend (nouvelle app)
✅ Préserve votre base de données (JAMAIS touchée)
✅ Régénère Prisma avec VOTRE base
✅ Sauvegarde tout avant de commencer

Temps: 2-3 minutes
Risque: ZERO (base de données non touchée)


MIGRATION DE LA BD (SI NECESSAIRE):
====================================

Utilisez: Scripts SQL manuels

Quand?
- Seulement si la nouvelle version nécessite des modifications
  de structure (nouvelles tables, colonnes, etc.)

Comment?
- Consultez: MIGRATIONS-BD.txt
- Appliquez les scripts SQL fournis
- Ou créez vos propres scripts

Temps: 1-2 minutes
Risque: FAIBLE (vous contrôlez tout)


EXEMPLE COMPLET:
================

Scénario: Mise à jour vers version 2.5 avec sessions de caisse

ETAPE 1: Migration du code
---------------------------

1. Placez le package dans le dossier client:
   
   📁 LOGESCO-Client-OPTIMISE\
   ├── 📁 backend\              ← Ancien
   ├── 📁 app\                  ← Ancien
   └── 📁 LOGESCO-Client-v2.5\  ← NOUVEAU PACKAGE

2. Copiez: MIGRATION-SIMPLE.bat dans le dossier client

3. Exécutez: MIGRATION-SIMPLE.bat

4. Suivez les 5 étapes affichées

5. Résultat:
   ✅ Nouveau code installé
   ✅ Base de données préservée
   ✅ Prisma régénéré


ETAPE 2: Migration BD (si nécessaire)
--------------------------------------

Consultez les notes de version:
"Version 2.5 nécessite la table cash_sessions"

1. Sauvegardez:
   copy backend\database\logesco.db backend\database\logesco_backup.db

2. Appliquez la migration:
   sqlite3 backend\database\logesco.db < backend\prisma\migrations\add_cash_sessions.sql

3. Vérifiez qu'il n'y a pas d'erreur

4. Démarrez l'application et testez


ETAPE 3: Test
-------------

1. Lancez: DEMARRER-LOGESCO.bat

2. Vérifiez:
   ✅ Vos produits s'affichent
   ✅ Vos clients s'affichent
   ✅ Vos ventes s'affichent
   ✅ Les nouvelles fonctionnalités marchent
