═══════════════════════════════════════════════════════════════
  ETAPE 1: CREER LE PACKAGE CORRIGE (Développeur uniquement)
═══════════════════════════════════════════════════════════════

Dans le dossier de développement:

  1. Double-clic sur: preparer-pour-client-CORRIGE.bat
  
  2. Attendez la fin (2-3 minutes)
  
  3. Vérifiez le message:
     ✅ Package CORRIGE (sans Prisma pre-genere, sans base vierge)
  
  4. Le package est créé dans:
     release\LOGESCO-Client-CORRIGE\
  
  5. VERIFICATION IMPORTANTE:
     Double-clic sur: VERIFIER-PACKAGE-CORRIGE.bat
     
     Vous devez voir:
     ✅✅✅ PACKAGE CORRECT ✅✅✅
  
  6. Copiez le dossier complet:
     release\LOGESCO-Client-CORRIGE\
     
     Sur une clé USB ou envoyez-le au client

  DEPANNAGE
═══════════════════════════════════════════════════════════════

PROBLEME: "Package CORRIGE non trouvé"
---------------------------------------

CAUSE: Le package n'est pas au bon endroit

SOLUTION:
  1. Vérifiez que vous êtes dans le dossier CLIENT
     (celui qui contient VOS données)
  
  2. Vérifiez la structure:
     
     📁 LOGESCO-Client-OPTIMISE\  ← Vous êtes ICI
     ├── 📁 backend\
     ├── 📁 LOGESCO-Client-CORRIGE\  ← Package ICI
     └── 📄 MIGRER-VERS-CORRIGE.bat  ← Script ICI
  
  3. Relancez le script


PROBLEME: "Base vierge détectée dans le package"
-------------------------------------------------

CAUSE: Le package contient une base de données vide

SOLUTION:
  1. Supprimez le fichier:
     LOGESCO-Client-CORRIGE\backend\database\logesco.db
  
  2. Relancez le script de migration


PROBLEME: "Produits: 0" après migration
----------------------------------------

CAUSE: La base n'a pas été restaurée correctement

SOLUTION:
  1. Vérifiez la sauvegarde:
     sauvegarde_migration_XXXXXX\logesco.db
  
  2. Copiez manuellement:
     De: sauvegarde_migration_XXXXXX\logesco.db
     Vers: backend\database\logesco.db
  
  3. Exécutez: REGENERER-PRISMA-SIMPLE.bat
  
  4. Redémarrez l'application


PROBLEME: Données visibles en ligne mais pas dans l'app
--------------------------------------------------------

CAUSE: Prisma n'est pas synchronisé

SOLUTION:
  1. Exécutez: REGENERER-PRISMA-SIMPLE.bat
  
  2. Vous devez voir: "Produits: 165"
  
  3. Redémarrez l'application
