========================================
  SOLUTION PROBLEME MIGRATION
  Base de Données Non Trouvée
========================================

PROBLÈME:
---------
1. Migration réussit mais données non affichées
2. Script dit "Base de données non trouvée" 
3. Backend affiche "produits: 165" mais dossier database vide

CAUSE:
------
- Le package contient une base VIERGE qui écrase vos données
- Prisma n'est pas synchronisé avec votre base
- Base de données dans un emplacement non-standard

SOLUTION EN 3 ÉTAPES:
---------------------

ÉTAPE 1: Copier les Scripts Corrigés
-------------------------------------
Depuis le dossier de développement, exécutez:

    copier-scripts-migration-vers-clients.bat

Cela copie automatiquement les scripts vers vos 2 clients.


ÉTAPE 2: Localiser la Base de Données
--------------------------------------
Allez dans le dossier du client concerné:

Pour Client 1:
    cd "C:\Users\DIGITAL MARKET\Videos\LOGESCO-Client-Ultimate"

Pour Client 2:
    cd "E:\Stage 2025\LOGESCO-Client-Optimise"

Puis exécutez:
    trouver-base-donnees.bat

Ce script va chercher logesco.db partout et vous dire où elle est.


ÉTAPE 3: Lancer la Migration Corrigée
--------------------------------------
Dans le même dossier, exécutez:

    migration-guidee-corrigee.bat

Ce script:
✅ Détecte automatiquement l'emplacement de votre base
✅ Supprime la base vierge du package
✅ Restaure VOS données
✅ Synchronise Prisma avec votre base
✅ Vérifie la configuration


SCRIPTS DISPONIBLES:
--------------------

1. copier-scripts-migration-vers-clients.bat
   → Copie tous les scripts vers les clients

2. trouver-base-donnees.bat
   → Localise logesco.db et affiche sa taille/contenu

3. migration-guidee-corrigee.bat
   → Migration intelligente avec détection automatique

4. verifier-config-database.bat
   → Vérifie .env et schema.prisma


VÉRIFICATION APRÈS MIGRATION:
------------------------------

1. Démarrer le backend:
   cd backend
   node src/server.js

   Vous devriez voir:
   produits: 165
   clients: X
   ventes: Y

2. Tester l'API:
   curl http://localhost:8080/api/v1/products

3. Lancer l'application et vérifier les données


SI ÇA NE FONCTIONNE PAS:
------------------------

1. Lisez: SOLUTION_PROBLEME_BASE_DONNEES.md
   (Guide complet avec diagnostic approfondi)

2. Restaurez l'ancien backend:
   rmdir /s /q backend
   ren backend_ancien backend

3. Contactez le support avec:
   - Résultat de trouver-base-donnees.bat
   - Résultat de verifier-config-database.bat
   - Messages d'erreur du backend


RÉSUMÉ RAPIDE:
--------------

1. copier-scripts-migration-vers-clients.bat
2. cd "dossier du client"
3. trouver-base-donnees.bat
4. migration-guidee-corrigee.bat
5. Tester!


IMPORTANT:
----------
- Exécutez TOUJOURS depuis le dossier d'installation du client
- Le script crée des sauvegardes automatiques
- En cas d'erreur, backend_ancien contient l'ancienne version
