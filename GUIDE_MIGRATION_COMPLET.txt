Le client doit avoir cette structure:

│   └── MIGRER-VERS-CORRIGE.bat

    ├── backend\

OU:

LOGESCO-Client-Optimise\         (client actuel)
├── backend\
├── app\
├── MIGRER-VERS-CORRIGE.bat
└── Package-Mise-A-Jour\
    └── LOGESCO-Client-CORRIGE\  (nouveau package)


═══════════════════════════════════════════════════════════════
  🚀 PROCÉDURE MIGRATION (CLIENT)
═══════════════════════════════════════════════════════════════

ÉTAPE 1: Recevoir le package
-----------------------------
Le client reçoit:
- LOGESCO-Client-CORRIGE (dossier)
- MIGRER-VERS-CORRIGE.bat (script)
- REGENERER-PRISMA-SIMPLE.bat (script de secours)


ÉTAPE 2: Placer les fichiers
-----------------------------
Option A: Dossier parent
  Copier LOGESCO-Client-CORRIGE dans le dossier parent

Option B: Sous-dossier
  Créer: Package-Mise-A-Jour\
  Copier LOGESCO-Client-CORRIGE dedans

Copier MIGRER-VERS-CORRIGE.bat dans le dossier client actuel


ÉTAPE 3: Exécuter la migration
-------------------------------
cd "C:\...\LOGESCO-Client-Optimise"
MIGRER-VERS-CORRIGE.bat


ÉTAPE 4: Suivre les instructions
---------------------------------
Le script guide étape par étape:
1. Vérification emplacement
2. Vérification base
3. Recherche package
4. Sauvegarde
5. Installation
6. Génération Prisma
7. Vérification


ÉTAPE 5: Tester
---------------
DEMARRER-LOGESCO.bat

Vérifier que les données s'affichent


═══════════════════════════════════════════════════════════════
  🔧 EN CAS DE PROBLÈME
═══════════════════════════════════════════════════════════════

Si après migration, les données ne s'affichent pas:

> REGENERER-PRISMA-SIMPLE.bat

Puis:

> DEMARRER-LOGESCO.bat
